# Korah Web — Agent Context

## Mission
Korah is a **100% free, open-source AI study app** built as a nonprofit passion project. The goal is to democratize access to high-quality AI-powered study tools for all students, regardless of background. No paywalls, no subscriptions, ever.

**GitHub org:** korah-repos (private during development, will be made public open-source)
**iOS companion app:** `korah-beta 2` (see its own AGENT.md)
**Shared backend:** Vercel serverless proxy at `https://korah-beta.vercel.app`

---

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | Plain HTML5, CSS3, Vanilla JavaScript (ES Modules) |
| CSS Framework | Tailwind CSS (CDN) |
| Reactive UI | AlpineJS 3.x (CDN) |
| Markdown | marked.js (CDN) |
| Math rendering | KaTeX (CDN) |
| Deployment | Vercel (static + serverless functions) |
| Backend | Node.js serverless functions (`/api/*.js`) |
| AI | OpenAI API (gpt-4o-mini, gpt-4o) via Vercel proxy |
| Rate limiting | Upstash Redis (via `@upstash/redis`) |

**No build step** — the web app is pure static HTML/CSS/JS. No bundler, no framework, no TypeScript.

## Current Architecture

### Chat App (`korah-bot/`)
- **Single-file IIFE pattern** in `korah-chat.js` — all state, rendering, and API calls are self-contained
- **localStorage** for all persistence: chat sessions (`korah_sessions`), study items (`korah_study_items`), settings (`korah_settings`)
- **Streaming SSE** responses from OpenAI via the Vercel proxy
- **Subject modes**: General, Math, Physics, Chemistry, Biology, History, Literature — each has its own system prompt, color theme, and welcome suggestions
- **Study items**: Flashcards, Study Guides, and Practice Tests generated by AI, stored in localStorage
- **No user accounts** on web (yet) — all data is device-local

### Serverless API (`korah-bot/api/`)
- `generate-study-item.js` — generates flashcards/study guides/practice tests from a prompt. Returns normalized JSON content shapes.
- The proxy itself lives in the `korah-beta 2` iOS repo (deployed as a shared Vercel project at `https://korah-beta.vercel.app`)

### Shared Backend (Cross-platform)
Both the web app and iOS app hit the same Vercel deployment:
- `POST /api/proxy` — OpenAI chat completions proxy (streaming + non-streaming)
- `POST /api/proxy-stream` — Dedicated streaming endpoint
- `POST /api/transcribe` — Whisper transcription
- `POST /api/speak` — Text-to-speech (TTS)
- `POST /api/generate-study-item` — AI study item generation
- Rate limiting via Upstash Redis (device ID header: `X-Device-ID`)

## Key Patterns & Conventions

### JavaScript
- **No build tools, no imports** — scripts are loaded via `<script>` tags or CDN
- **IIFE pattern** for the main chat module: `(() => { ... })()`
- **Storage utilities** live in the `Storage` object (see `korah-chat.js` lines 24–104)
- **Streaming** uses `ReadableStream` + `TextDecoder` to parse OpenAI SSE chunks
- Never use `var` — use `const` or `let`
- Always handle errors gracefully with user-facing messages

### CSS
- **Tailwind utility classes** for layout and spacing
- **Custom CSS variables** for theming: `--p4`, `--p5`, `--ac`, `--glow` are overridden per subject mode
- **Dark/light mode** via `data-theme` attribute toggled by AlpineJS
- CSS class naming: functional names like `.msg-bubble`, `.history-item`, `.suggestion-chip`

### HTML
- AlpineJS `x-data` on the root `<html>` element for theme management
- IDs for JS-targeted elements, classes for styling
- Keep modals inline in the HTML, toggled by `.show` class

## Planned: OpenNotebook Features

The primary development goal is implementing NotebookLM-inspired features. Reference the full plan for details. Below is the high-level roadmap:

### Phase 1 — Backend Foundation
- Add a lightweight Node.js/Express (or Python FastAPI) backend inside `korah-bot/backend/` or `korah-bot/server/`
- SQLite database with tables: `notebooks`, `sources`, `chunks`, `embeddings`, `notes`, `chat_sessions`
- File upload endpoint accepting PDF, TXT, DOCX, MD
- Environment variables: `OPENAI_API_KEY`, `DATABASE_PATH`, `UPLOAD_FOLDER`

### Phase 2 — Document Processing & RAG
- Text chunking (~500–1000 tokens with overlap)
- OpenAI `text-embedding-3-small` for vector embeddings
- Cosine similarity search for relevant context retrieval

### Phase 3 — Enhanced Chat with RAG
- Context injection before OpenAI calls using retrieved source chunks
- Citation tracking (which sources contributed to a response)
- Session stored in DB instead of localStorage

### Phase 4 — Frontend Enhancements
- Notebook selector in sidebar
- Source upload panel (drag-and-drop or file picker)
- Notes panel (manual + AI-generated)
- Semantic search interface

### Phase 5 — Content Transformations
- Quick-action buttons: Summarize, Key Points, Study Guide, Flashcards, Explain Simply
- `POST /api/transformations/execute` endpoint
- Integrate with existing subject modes

### Phase 6 — Optional Advanced Features
- Audio/video file transcription (Whisper)
- Web page import via URL
- Basic podcast generation (OpenAI TTS)

## Open-Source References

These repositories live in `../open-source/` relative to this repo. AI agents should research and adapt code from them when implementing features.

### `open-source/open-notebook/`
The **primary reference implementation** for NotebookLM features. Full-stack app:
- **Backend**: Python FastAPI (`open_notebook/`) — study notebooks, sources, vector embeddings, RAG chat, content transformations, podcast generation
- **Frontend**: Next.js (`frontend/`) — notebook management, source uploading, chat with context, notes
- **Database**: SurrealDB (we will adapt to SQLite)
- **Key paths to reference**:
  - `open_notebook/` — core Python logic for embeddings, chunking, transformations
  - `frontend/` — UI patterns for notebooks/sources/notes
  - `api/` — REST API structure
  - `prompts/` — AI prompt templates for transformations
- Use this as a **conceptual and code reference** — do not copy wholesale, adapt to our simpler JS/Node.js stack

### `open-source/react-quizlet-flashcard/`
React flashcard component with flip animations and deck navigation (`FlashcardArray`, `Flashcard`). Reference for improving the web flashcard UI:
- Smooth CSS 3D flip animations
- Progress tracking
- Keyboard navigation
- Accessible ARIA attributes
- Source: `package/` directory contains the library code

### `open-source/OpenSAT/`
Free SAT question database (Python Flask + JSON):
- Public API at `https://pinesat.com/api/questions`
- Questions include domain, paragraph, choices, correct answer, explanation
- Reference for: integrating a practice question bank, question object schema, subject-specific test generation
- Useful for a future "SAT Practice Mode" in korah-bot

## Development Rules

1. **Never expose the OpenAI API key in frontend code** — always proxy through Vercel serverless functions
2. **Preserve existing chat UX** — new features should be additive, not disruptive
3. **Progressive enhancement** — the app should work even if new features fail gracefully
4. **Mobile-first** — test all UI changes at 375px width
5. **No new CDN dependencies without discussion** — keep the bundle lean
6. **localStorage keys** — always prefix with `korah_` to avoid collisions
7. **Do not build/compile** — there is no build step; changes are live immediately
8. The web app and iOS app share design language (purple theme `#8b5cf6`, dark backgrounds, glass effects). Keep them visually consistent.
